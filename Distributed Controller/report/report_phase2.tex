\section{Introduction}

This report describes the second stage of the project, where three smart luminaires were integrated into a cooperative distributed illumination system. Compared with the local controller developed in Phase~1, this stage adds CAN-BUS communications, a hub interface between the PC and the control network, a wake-up and calibration procedure executed at start-up, and distributed decision-making algorithms.

The objective is to minimise total energy consumption while keeping desk illuminance above the visibility lower bounds associated with the occupation states LOW and HIGH. Since each luminaire affects not only its own desk but also neighbouring desks, the control problem is coupled and must be solved cooperatively.

% ===================================================================
\section{System Architecture}

The experimental setup consists of a closed office-scale box containing three luminaires. Each node contains one RP2040-based microcontroller (Raspberry Pi Pico), one LED driven by PWM, one LDR-based luxmeter, and one MCP2515 CAN controller interfaced via SPI. The LED and the LDR are mounted so that the sensor measures reflected light on the desk surface.

Any node connected to the PC through USB serial can operate as a hub, forwarding commands to the CAN network and returning measurements. The command interface supports duty-cycle setting, lux reference updates, occupancy state changes, enable/disable flags, and performance queries.

All nodes run identical firmware. The software is organised as a non-blocking concurrent application on the RP2040 dual-core architecture:
\begin{itemize}[nosep]
  \item \textbf{Core~0}: control state machines, serial parsing, hub function, calibration logic, PI controller, and distributed coordination;
  \item \textbf{Core~1}: MCP2515 CAN controller service exclusively --- interrupt handling, frame TX/RX, and low-latency transfer via FIFO queues.
\end{itemize}

This separation keeps the application responsive and avoids blocking interactions between communications and control. The control loop runs at 100\,Hz (10\,ms period), while CAN frame processing on Core~1 runs asynchronously.

% ===================================================================
\section{CAN-BUS Communication}

The nodes communicate through standard 11-bit CAN frames at 500\,kbps. The MCP2515 modules use 16\,MHz crystals and are powered at 5\,V (VBUS). The CAN bus is terminated with 120\,$\Omega$ resistors on the two end nodes. A common ground connection between all modules is required in addition to the USB ground.

The protocol supports both unicast and broadcast messages. The main message classes are:
\begin{itemize}[nosep]
  \item Node discovery and hello frames (wake-up phase);
  \item Hub commands forwarded from the PC;
  \item Query/reply messages for the command-line interface;
  \item Calibration plan broadcast and slot synchronisation;
  \item Distributed control data exchange (gain rows, proposals, dual variables).
\end{itemize}

The hub node translates serial requests from the PC into CAN messages and routes replies back in the textual format expected by the command-line interface. This allows the complete network to be operated from a single terminal.

% ===================================================================
\section{Network Boot \& Calibration}

At power-on, all nodes enter a discovery phase where they announce their presence via CAN and learn which neighbours are active. After discovery stabilises, the network starts a synchronous calibration procedure that estimates the system model parameters.

The system model is:
\begin{equation}
  \mathbf{L} = \mathbf{K} \cdot \mathbf{d} + \mathbf{o}
  \label{eq:model}
\end{equation}
where $\mathbf{L} \in \mathbb{R}^N$ is the illuminance vector (lux), $\mathbf{d} \in [0,1]^N$ is the duty cycle vector, $\mathbf{K} \in \mathbb{R}^{N \times N}$ is the gain matrix (lux per unit duty), and $\mathbf{o} \in \mathbb{R}^N$ is the external (background) illuminance.

The calibration procedure follows three steps:
\begin{enumerate}[nosep]
  \item All LEDs off: measure the background illuminance $\mathbf{o}$;
  \item For each luminaire $i$: turn on LED~$i$ at a known duty cycle (50\%), all others off, and measure the resulting illuminance at all desks;
  \item Compute gains: $K_{ji} = (L_j^{(i)} - o_j) / d_{\mathrm{cal}}$.
\end{enumerate}

The calibration plan is broadcast on CAN so all nodes follow the same schedule. Each phase uses a 250\,ms settling time and 600\,ms measurement window. For $N=3$ nodes, the total calibration time is approximately 4.2\,seconds:
\begin{equation}
  T_{\mathrm{cal}} = 850 + N \times 1100 \;\text{ms} \approx 4150\;\text{ms}
  \label{eq:cal_time}
\end{equation}

The measured gain matrix and background vector for our setup are:

\section{Local Controller}

Each node runs a PI controller with feedforward for fast reference tracking:
\begin{equation}
  u(t) = u_{\mathrm{ff}} + K_p \, e(t) + K_i \int_0^t e(\tau)\,d\tau
  \label{eq:pi}
\end{equation}
where the feedforward term provides an immediate open-loop estimate:
\begin{equation}
  u_{\mathrm{ff}} = \frac{L_{\mathrm{ref}} - o_i}{K_{ii}}
  \label{eq:ff}
\end{equation}

The PI gains are $K_p = 0.5$ and $K_i = 0.3$, with anti-windup (back-calculation) to prevent integral saturation. The feedforward term allows the controller to reach the target illuminance within the first control cycle, with the PI correcting residual errors from model mismatch and external disturbances.

% ===================================================================
\section{Distributed Control Algorithms}

The distributed optimisation problem is:
\begin{equation}
  \min_{\mathbf{d}} \sum_{i=1}^{N} c_i \, d_i
  \quad \text{s.t.} \quad
  \mathbf{K}\mathbf{d} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}, \;
  0 \leq d_i \leq 1
  \label{eq:opt}
\end{equation}
where $c_i$ is the energy cost coefficient for luminaire~$i$. This formulation minimises weighted energy consumption while satisfying visibility constraints at all desks.

Three distributed algorithms were implemented and compared against the local PI baseline.

\subsection{Consensus-Based Optimisation}

Each node $i$ solves a local sub-problem:
\begin{equation}
  \min_{d_i} \; c_i d_i + \frac{\rho}{2}(d_i - \bar{d}_i)^2
  \quad \text{s.t.} \quad \mathbf{K}_{:,i} d_i + \text{others} \geq \mathbf{L}_{\text{ref}}
\end{equation}
where $\bar{d}_i$ is the current consensus average. The unconstrained optimum is $d_i^* = \bar{d}_i - c_i/\rho$, which is then projected onto the feasible set defined by the illuminance constraints. Proposals are exchanged via CAN and averaged across all nodes. The penalty parameter is $\rho = 2.0$.

\subsection{ADMM}

The Alternating Direction Method of Multipliers decomposes the problem into:
\begin{enumerate}[nosep]
  \item \textbf{d-update}: $d_i = z_i - u_i - c_i/\rho$ (local primal);
  \item \textbf{z-update}: project $\mathbf{z} = \mathbf{d} + \mathbf{u}$ onto the constraint set $\mathbf{K}\mathbf{z} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}$ using iterative Dykstra projections;
  \item \textbf{u-update}: $u_i \leftarrow u_i + d_i - z_i$ (dual variable).
\end{enumerate}
The penalty parameter is $\rho = 1.0$. Convergence is detected when the primal residual $\|\mathbf{d} - \mathbf{z}\|$ falls below $10^{-3}$.

\subsection{Dual Decomposition}

The Lagrangian relaxation approach uses gradient descent on the primal and ascent on the dual:
\begin{align}
  d_i &\leftarrow d_i - \alpha \left( c_i - \sum_k \lambda_k K_{ki} \right) \\
  \lambda_k &\leftarrow \max\left(0,\; \lambda_k + \alpha (L_{\text{ref},k} - \mathbf{K}_{k,:}\mathbf{d} - o_k)\right)
\end{align}
The step size $\alpha$ starts at 0.05 and decays by a factor of 0.995 per iteration to ensure convergence. The dual variables $\lambda_k$ are shared via CAN.

% ===================================================================
\section{Performance Metrics}

Three metrics are computed continuously at each control cycle:

\textbf{Energy} $E$ is the accumulated electrical energy:
\begin{equation}
  E = \sum_{k} P(k) \cdot \Delta t
\end{equation}
where $P(k)$ is the instantaneous power at sample $k$ and $\Delta t = 10$\,ms.

\textbf{Visibility error} $V$ is the time-averaged illuminance deficit:
\begin{equation}
  V = \frac{1}{N_s} \sum_{k} \max(0, \; L_{\text{ref}} - L(k))
\end{equation}

\textbf{Flicker} $F$ captures duty-cycle oscillations via changes in direction:
\begin{equation}
  F = \frac{1}{N_s} \sum_{k} |d(k) - d(k-1)| + |d(k-1) - d(k-2)|
\end{equation}

A dedicated \texttt{RM} command resets all metrics to zero, enabling fair per-algorithm comparisons.

\section{Results}

\subsection{Calibration}

The wake-up and calibration procedure completes in approximately 4.2\,seconds for three nodes. Table~\ref{tab:K} shows the measured gain matrix. The diagonal dominance confirms that each LED primarily illuminates its own desk, with cross-coupling gains 2--80$\times$ weaker than the self-coupling. This physical characteristic limits the degree to which distributed algorithms can redistribute duty based on energy costs.

\subsection{Unequal Energy Costs}

With costs $c = [1, 2, 3]$, the distributed algorithms should ideally shift duty towards the cheapest node (Node~1). However, the measured gain matrix shows that cross-coupling is weak relative to self-coupling: even at maximum duty, Node~1's LED contributes less than 1\,lux to Node~2's sensor ($K_{21} \times 1.0 = 0.89$\,lux). This means each node is largely constrained to provide its own illuminance, leaving little room for cost-based redistribution.

The differences are marginal ($<$2\% change in duty). This is not a limitation of the algorithm but of the physical geometry: the cross-coupling gains are 10--80$\times$ weaker than the self-coupling (e.g., $K_{21}/K_{22} = 0.012$). Even at maximum duty, Node~1's LED contributes less than 1\,lux to Node~2's sensor. Each node is therefore largely constrained to provide its own illuminance, leaving minimal room for cost-based redistribution. In a system with stronger inter-desk coupling, the optimisation would produce more pronounced duty differentiation.

ADMM and dual decomposition were also implemented and tested. While both converged correctly in isolated tests, they showed sensitivity to algorithm switching during rapid automated testing (residual state from previous algorithm modes). In practice, the consensus approach proved the most robust for online switching.

\section{Conclusions}

The distributed illumination system satisfies the main objectives of Phase~2: CAN-BUS communications at 500\,kbps, hub-based PC interaction, deterministic network wake-up, automated calibration of the gain matrix, and distributed cooperative control.

The PI controller with feedforward provides fast reference tracking (sub-second settling) and effective disturbance rejection. Three distributed optimisation algorithms (Consensus, ADMM, and Dual Decomposition) were implemented and verified to converge to feasible operating points that satisfy all illuminance constraints.

With equal energy costs, all algorithms converge to the same solution, as expected from the unique feasible point of the linear program. With unequal costs, the redistribution effect is limited by the weak cross-coupling in the physical setup. In a system with stronger inter-desk coupling, the cost-based optimisation would produce more pronounced duty differentiation.

The key design decisions that proved important were: (i)~dedicating Core~1 exclusively to CAN communication, avoiding SPI contention; (ii)~using feedforward for immediate open-loop response, reducing the burden on the PI integrator; and (iii)~broadcasting the calibration plan over CAN for deterministic multi-node synchronisation.
