\section{Introduction}

This report presents the distributed cooperative illumination system developed in Phase~2. Three smart luminaires communicate over CAN-BUS, automatically calibrate a linear model at start-up, and run distributed optimisation algorithms to minimise energy while maintaining desk illuminance above occupancy-dependent lower bounds. Compared with the local PI controller of Phase~1, this stage adds inter-node communication, cooperative decision-making, and a hub interface for PC control. Since each luminaire affects not only its own desk but also neighbouring desks through cross-coupling, the control problem is coupled and must be solved cooperatively.

\section{System Architecture}

Each node comprises an RP2040 Pico, an LED (12-bit PWM), an LDR luxmeter (10\,k$\Omega$ divider), and an MCP2515 CAN controller (SPI, 16\,MHz crystal, 5\,V supply). LEDs are mounted in 3D-printed fixtures that minimise external light seepage.

\section{CAN-BUS Communication}

The CAN bus operates at 500\,kbps with standard 11-bit identifiers, terminated with 120\,$\Omega$ resistors. The ID encodes message type plus sender/target: $\texttt{CAN\_ID} = \texttt{BASE} + \texttt{node\_id}$. Lower IDs win arbitration, so management traffic (Hello, 0x100) always preempts algorithm data (0x700+). Broadcasts use address \texttt{0x7F}. Table~\ref{tab:can} lists all ten message types.

The system model is $\mathbf{L} = \mathbf{K}\mathbf{d} + \mathbf{o}$, where $\mathbf{K}$ is the gain matrix and $\mathbf{o}$ the background. Calibration measures these via round-robin: baseline (all LEDs off), then each LED at 50\% duty while all sensors record, yielding $K_{ji} = (L_j^{(i)} - o_j)/d_{\text{cal}}$. The state machine (Figure~\ref{fig:calib_sm}) completes in ${\sim}4150$\,ms.

\section{Local Controller}

Each node runs a discrete PI controller with feedforward at 100\,Hz (10\,ms period). The feedforward term $u_{\text{ff}} = (L_{\text{ref}} - o_i)/K_{ii}$ inverts the calibrated model to provide an immediate open-loop duty estimate, enabling single-cycle reference tracking. The PI component ($K_p{=}0.5$, $K_i{=}0.3$) corrects residual errors from model mismatch and external disturbances. Back-calculation anti-windup prevents integrator saturation when the duty cycle is clamped at 0 or 1.

\section{Distributed Algorithms}

The optimisation minimises $\sum c_i d_i$ subject to $\mathbf{K}\mathbf{d} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}$ and $0 \leq d_i \leq 1$. Three algorithms were implemented:

\textbf{Consensus ADMM.} Each node solves $\min c_i d_i + (\rho/2)(d_i - \bar{d}_i)^2$ subject to all illuminance constraints, exchanges proposals via CAN, and averages ($\rho{=}2.0$, $k_{\max}{=}50$).

\textbf{ADMM.} Decomposes into primal $d_i = z_i - u_i - c_i/\rho$, feasibility projection $\mathbf{z} = \Pi_{\mathcal{C}}(\mathbf{d}+\mathbf{u})$, and dual update $u_i \mathrel{+}= d_i - z_i$ ($\rho{=}1.0$).

\textbf{Dual Decomposition.} Subgradient ascent on Lagrange multipliers with step decay ($\alpha_0{=}0.05$, decay 0.995).

\section{Results}

\subsection{Algorithm Comparison}

All four algorithms were tested with equal costs $c{=}[1,1,1]$, $L_{\text{ref}}{=}20$\,lux, and metrics reset before each 20-second window. All converge to the same operating point (Table~\ref{tab:equal}), confirming the unique feasible minimum. Visibility error is below 0.026\,lux and flicker is zero in steady state, with total energy ${\sim}28.3$\,J. Consensus proved most robust during testing; ADMM and dual decomposition showed sensitivity to rapid algorithm switching.

\section{Conclusions}

The system achieves all Phase~2 objectives: 500\,kbps CAN communication, hub-based PC control, deterministic wake-up, automated calibration (${\sim}4.2$\,s), and distributed cooperative control. The PI controller with feedforward provides sub-100\,ms reference tracking and effective disturbance rejection. Three optimisation algorithms were implemented and validated: with equal costs all converge to the same feasible point; with unequal costs, redistribution is limited by the weak physical cross-coupling (each LED dominates its own sensor by 7--70$\times$). Key design choices include: (i)~Core~1 exclusively for CAN, eliminating SPI contention; (ii)~feedforward initialisation for single-cycle tracking; (iii)~CAN-broadcast calibration plans for deterministic synchronisation; (iv)~full-constraint checking in the local optimisation, where each node considers all peers' illuminance requirements.
