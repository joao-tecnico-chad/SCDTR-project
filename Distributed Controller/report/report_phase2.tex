\section{Introduction}

This report describes the design and implementation of a distributed cooperative illumination control system.
Building on the local PI controller developed in Phase~1, Phase~2 adds CAN-BUS networking between three luminaire nodes, a hub function for PC interfacing, coordinated system calibration, and three distributed optimisation algorithms: consensus, dual decomposition, and ADMM.

The objective is to minimise global energy consumption while maintaining illuminance above desk-specific thresholds, considering both self and cross-coupling effects between luminaires.

% ===================================================================
\section{System Architecture}

\subsection{Hardware Setup}

Three RPI Pico W microcontrollers, each driving an LED and reading an LDR, are connected via CAN-BUS using MCP2515 modules.
One node, connected to the PC via USB serial, acts as the hub, bridging PC commands to the CAN network.

\subsection{Software Architecture}

All nodes run identical firmware. The RP2040 dual-core architecture is exploited:

\begin{itemize}[nosep]
  \item \textbf{Core~1}: Local PI control loop at 100\,Hz (same as Phase~1).
  \item \textbf{Core~0}: Serial/CAN message handling, hub function, and distributed algorithm iteration at 10\,Hz.
\end{itemize}

The distributed algorithm computes optimal duty cycles, which feed into the local PI controller as illuminance references.

% ===================================================================
\section{CAN-BUS Communication}

Messages use the standard CAN frame format (11-bit arbitration ID, 8-byte payload).
The arbitration ID encodes the sender and message type:
\[
  \text{ID} = (\text{sender} \ll 4) \;\mathbin{|}\; \text{msg\_type}
\]

Message types include boot announcements, calibration data, duty/reference updates, and hub commands.

% ===================================================================
\section{Network Boot \& Calibration}

\subsection{Node Discovery}

On power-up, each node broadcasts its unique hardware ID (from the Pico flash) for 3\,s.
All discovered IDs are sorted deterministically; the rank in the sorted list becomes the logical ID (1--3).
This ensures identical ID assignment regardless of boot order.

\subsection{System Calibration}

After boot, a coordinated calibration sequence measures the gain matrix $\mathbf{K}$ and background vector $\mathbf{o}$:

\begin{enumerate}[nosep]
  \item All LEDs off: measure background $o_i$ at each node.
  \item Turn on node~$j$ at full duty, others off: measure $K_{ij}$ at all nodes.
  \item Repeat for each node.
\end{enumerate}

The system model is:
\begin{equation}
  \mathbf{L} = \mathbf{K} \cdot \mathbf{d} + \mathbf{o}
  \label{eq:model}
\end{equation}
where $\mathbf{L}$ is the illuminance vector, $\mathbf{d}$ the duty cycle vector, and $\mathbf{o}$ the external illuminance.

\section{Distributed Control Algorithms}

The optimisation problem is:
\begin{equation}
  \min_{\mathbf{d}} \sum_{i=1}^{N} c_i \, d_i
  \quad \text{s.t.} \quad
  \mathbf{K}\mathbf{d} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}, \;
  0 \leq d_i \leq 1
  \label{eq:opt}
\end{equation}
where $c_i$ is the energy cost coefficient for luminaire~$i$.

Three algorithms are implemented, switchable at runtime via serial command.

\subsection{Consensus}

Each node solves a local subproblem incorporating a penalty term for deviating from the network average.
Duty cycle proposals are exchanged via CAN-BUS and averaged each iteration.
Convergence is typically achieved in 10--50 iterations.

\subsection{Dual Decomposition}

The constraints are relaxed using Lagrange multipliers $\lambda_i$.
Each node updates its duty by minimising the local Lagrangian; multipliers are updated via subgradient ascent.
A decaying step size $\alpha$ ensures convergence.

\subsection{ADMM}

ADMM splits the problem into local d-updates, a global z-projection onto the constraint set, and dual u-updates.
The penalty parameter $\rho$ controls the trade-off between primal and dual convergence.
