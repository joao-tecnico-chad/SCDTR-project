\section{Introduction}

This report describes the second stage of the project, where three smart luminaires were integrated into a cooperative distributed illumination system. Compared with the local controller developed in Phase~1, this stage adds CAN-BUS communications, a hub interface between the PC and the control network, a wake-up and calibration procedure executed at start-up, and distributed decision-making algorithms.

The objective is to minimise total energy consumption while keeping desk illuminance above the visibility lower bounds associated with the occupation states LOW and HIGH. Since each luminaire affects not only its own desk but also neighbouring desks, the control problem is coupled and must be solved cooperatively.

The system model assumes a linear relationship between duty cycles and illuminance, calibrated automatically at start-up. Three distributed optimisation algorithms---consensus ADMM, ADMM, and dual decomposition---are compared against a local PI baseline.

\section{System Architecture}

\subsection{Hardware}

Each node contains one RP2040-based microcontroller (Raspberry Pi Pico), one LED driven by 12-bit PWM (0--4095), one LDR-based luxmeter with a voltage-divider circuit (10\,k$\Omega$ fixed resistor), and one MCP2515 CAN controller interfaced via SPI at 10\,MHz. The LED and the LDR are mounted so that the sensor measures reflected light on the desk surface.

The MCP2515 modules use 16\,MHz crystals and are powered at 5\,V from the Pico VBUS pin. A common ground between all modules is essential for reliable communication.

\section{CAN-BUS Communication}

\subsection{Bus Configuration}

The CAN bus operates at 500\,kbps using standard 11-bit identifiers. The bus is terminated with 120\,$\Omega$ resistors on the two end nodes.

\subsection{Message Protocol}

The 11-bit CAN identifier is composed of a base address that encodes the message type plus a node offset that identifies the sender or target:

\begin{equation}
  \texttt{CAN\_ID} = \texttt{BASE} + \texttt{node\_id}
  \label{eq:canid}
\end{equation}

Table~\ref{tab:can} lists all message types. Broadcast messages use a reserved node address (\texttt{0x7F}).

\subsection{System Model}

The illuminance model is:
\begin{equation}
  \mathbf{L} = \mathbf{K} \cdot \mathbf{d} + \mathbf{o}
  \label{eq:model}
\end{equation}
where $\mathbf{L} \in \mathbb{R}^N$ is the illuminance vector, $\mathbf{d} \in [0,1]^N$ the duty cycle vector, $\mathbf{K} \in \mathbb{R}^{N \times N}$ the gain matrix, and $\mathbf{o} \in \mathbb{R}^N$ the background (ambient) illuminance.

Calibration proceeds as a round-robin over all $N$ nodes:
\begin{enumerate}[nosep]
  \item \textbf{Baseline}: all LEDs off, measure background $\mathbf{o}$ (250\,ms settle + 600\,ms measure);
  \item \textbf{Slot $i$}: LED~$i$ at 50\% duty, others off---measure $L_j^{(i)}$ at all sensors (250\,ms settle + 600\,ms measure);
  \item \textbf{Compute}: $K_{ji} = (L_j^{(i)} - o_j) / d_{\text{cal}}$.
\end{enumerate}

After the last slot, each node broadcasts its gain row, cost coefficient, and reference via the Gain Exchange message. Total calibration time for $N{=}3$:
\begin{equation}
  T_{\text{cal}} = 850 + N \times 1100 \approx 4150\;\text{ms}
  \label{eq:tcal}
\end{equation}

\section{Local Controller}

Each node runs a PI controller with feedforward:
\begin{equation}
  u(t) = u_{\text{ff}} + K_p \, e(t) + K_i \int_0^t e(\tau)\,d\tau
  \label{eq:pi}
\end{equation}
where the feedforward provides an immediate open-loop estimate:
\begin{equation}
  u_{\text{ff}} = \frac{L_{\text{ref}} - o_i}{K_{ii}}
  \label{eq:ff}
\end{equation}

The PI gains are $K_p = 0.5$ and $K_i = 0.3$, with anti-windup via back-calculation. The feedforward allows the controller to reach the target illuminance within one control cycle; the PI corrects residual errors from model mismatch and disturbances.

\section{Distributed Control Algorithms}

The optimisation problem is:
\begin{equation}
  \min_{\mathbf{d}} \sum_{i=1}^{N} c_i \, d_i
  \quad \text{s.t.} \quad
  \mathbf{K}\mathbf{d} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}, \;
  0 \leq d_i \leq 1
  \label{eq:opt}
\end{equation}
where $c_i$ is the energy cost coefficient for luminaire~$i$. Three distributed algorithms were implemented and compared against the local PI baseline.

\subsection{Consensus ADMM}

Each node $i$ maintains a local copy of the full duty vector and solves:
\begin{equation}
  \min_{d_i} \; c_i d_i + \frac{\rho}{2}(d_i - \bar{d}_i)^2
  \quad \text{s.t.} \quad \text{all illuminance constraints}
  \label{eq:cons}
\end{equation}

The unconstrained optimum is $d_i^* = \bar{d}_i - c_i/\rho$, projected onto the feasible set defined by all $N$ illuminance constraints. Each node checks that its duty satisfies not only its own constraint but also the constraints of all peers (using the shared gain matrix). Proposals are exchanged via CAN and averaged. Parameters: $\rho = 2.0$, $k_{\max} = 50$ iterations.

\subsection{ADMM}

The Alternating Direction Method of Multipliers decomposes~\eqref{eq:opt} into primal, consensus, and dual updates:
\begin{align}
  d_i^{(k+1)} &= z_i^{(k)} - u_i^{(k)} - c_i/\rho  \label{eq:admm_d}\\
  \mathbf{z}^{(k+1)} &= \Pi_{\mathcal{C}}\!\left(\mathbf{d}^{(k+1)} + \mathbf{u}^{(k)}\right)  \label{eq:admm_z}\\
  u_i^{(k+1)} &= u_i^{(k)} + d_i^{(k+1)} - z_i^{(k+1)}  \label{eq:admm_u}
\end{align}
where $\Pi_{\mathcal{C}}$ projects onto $\mathbf{K}\mathbf{z} + \mathbf{o} \geq \mathbf{L}_{\text{ref}}$ using iterative Dykstra projections ($\rho = 1.0$).

\subsection{Dual Decomposition}

The Lagrangian relaxation uses subgradient ascent on the dual variables:
\begin{align}
  d_i &\leftarrow d_i - \alpha \left( c_i - \textstyle\sum_k \lambda_k K_{ki} \right) \label{eq:dd_d}\\
  \lambda_k &\leftarrow \max\!\left(0,\; \lambda_k + \alpha (L_{\text{ref},k} - \mathbf{K}_{k,:}\mathbf{d} - o_k)\right) \label{eq:dd_lam}
\end{align}
with $\alpha_0 = 0.05$ and decay factor 0.995 per iteration to ensure convergence.

\section{Performance Metrics}

Three metrics are computed at each 10\,ms control cycle:

\textbf{Energy:} $E = \sum_k P(k) \cdot \Delta t$, where $P(k)$ is instantaneous power proportional to duty.

\textbf{Visibility error:} $V = \frac{1}{N_s} \sum_k \max(0, L_{\text{ref}} - L(k))$, measuring the average illuminance deficit below the reference.

\textbf{Flicker:} $F = \frac{1}{N_s} \sum_k (|d_k - d_{k-1}| + |d_{k-1} - d_{k-2}|)$, quantifying rapid duty-cycle oscillations.

A dedicated \texttt{RM} command resets all metric accumulators, enabling fair per-algorithm comparisons without restarting the system.

\textbf{Programmatic disturbance.} To measure recovery speed without the slow polling bottleneck of the physical test, Node~2's duty was forced to 0.8 (open-loop) while Node~1 was observed at high time resolution (Figure~\ref{fig:dist_prog}). Node~2's cross-illumination raises Node~1's lux, so the PI controller reduces Node~1's duty from 0.52 to 0.15 (a 70\% reduction) to maintain the target. When Node~2's feedback is restored, Node~1 recovers within one polling sample ($<$1.3\,s measurement resolution). The actual PI recovery is sub-second, consistent with the feedforward-assisted step response.

\section{Conclusions}

The distributed illumination system satisfies the main objectives of Phase~2: CAN-BUS communications at 500\,kbps, hub-based PC interaction, deterministic network wake-up, automated calibration ($\sim$4.2\,s for 3 nodes), and distributed cooperative control.

The PI controller with feedforward provides fast reference tracking (sub-100\,ms settling) and effective disturbance rejection (sub-second recovery, confirmed via programmatic disturbance testing). Three distributed optimisation algorithms were implemented. With equal costs, all converge to the same feasible point, validating correctness. With unequal costs, redistribution is limited by the weak cross-coupling inherent to the physical setup---each node's LED dominates its own sensor by a factor of 7--70$\times$ over neighbours.

Key design decisions and their rationale:
\begin{enumerate}[nosep]
  \item \textbf{Core~1 exclusively for CAN}: eliminates SPI contention between control loop and communications;
  \item \textbf{Feedforward initialisation}: achieves single-cycle reference tracking, reducing PI integrator load;
  \item \textbf{CAN-broadcast calibration plan}: all nodes share a deterministic schedule, avoiding coordination overhead;
  \item \textbf{Metrics reset command}: enables fair per-algorithm performance comparison without system restart;
  \item \textbf{Full-constraint checking}: each node's local optimisation considers all peers' illuminance constraints, not just its own.
\end{enumerate}

\subsection{Future Work}

Potential improvements include: (i)~stronger physical cross-coupling (closer LED/sensor placement) to better demonstrate cost-based duty redistribution; (ii)~adaptive $\rho$ tuning for ADMM/consensus to improve convergence speed; (iii)~a web-based dashboard replacing the serial hub for real-time monitoring; and (iv)~power metering with actual current sensors instead of duty-based estimation.

\section{Optimisation Formulation}

\textbf{Augmented Lagrangian (Consensus):}
\begin{equation}
  L_\rho(\mathbf{x}, \bar{\mathbf{u}}, \boldsymbol{\lambda}) = \sum_{i=1}^{N}\left(f_i(\mathbf{x}_i) + \boldsymbol{\lambda}_i^T(\mathbf{x}_i - \bar{\mathbf{u}}) + \frac{\rho}{2}\|\mathbf{x}_i - \bar{\mathbf{u}}\|^2\right)
\end{equation}

\textbf{Primal update} (per node $i$):
\begin{equation}
  \mathbf{x}_i^{(k+1)} := \arg\min_{\mathbf{x}_i \in \mathcal{C}_i} \left\{ f_i(\mathbf{x}_i) + \boldsymbol{\lambda}_i^{(k)T}\mathbf{x}_i + \frac{\rho}{2}\|\mathbf{x}_i - \bar{\mathbf{u}}^{(k)}\|^2 \right\}
\end{equation}
where $\mathcal{C}_i = \{\mathbf{d} : \mathbf{K}\mathbf{d} + \mathbf{o} \geq \mathbf{L}_{\text{ref}},\; 0 \leq d_j \leq 1\}$.

\textbf{Consensus update:}
\begin{equation}
  \bar{\mathbf{u}}^{(k+1)} = \frac{1}{N}\sum_{i=1}^{N} \mathbf{x}_i^{(k+1)}
\end{equation}

\textbf{Dual update:}
\begin{equation}
  \boldsymbol{\lambda}_i^{(k+1)} := \boldsymbol{\lambda}_i^{(k)} + \rho(\mathbf{x}_i^{(k+1)} - \bar{\mathbf{u}}^{(k+1)})
\end{equation}
